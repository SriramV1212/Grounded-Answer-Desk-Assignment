Grounded Answer Desk

Your system answers questions from a body of knowledge — and admits when it doesn't know. Stand up an assistant that retrieves from a corpus you ingest and answers with citations, where your OpenClaw or Hermes agent reaches retrieval through an MCP server you build.

Why this assignment

An LLM that answers from memory bluffs. The job is grounding: retrieve the right passages, answer ONLY from them, cite, and abstain when the corpus doesn't cover it. This is the core test — do you understand RAG (the retrieval) AND MCP (the tool boundary your agent retrieves through) and how they fit together.

Use any AI coding tools (Claude Code, Cursor, etc.) — that's expected. We grade architecture and judgement: clean retrieval, a real MCP tool boundary, honest grounding — not lines of code. Aim for production-grade — clean, reliable, something a teammate could actually run.

The brief

  1. An OpenClaw or Hermes agent on a free-tier VPS you control.
  2. Ingest a corpus you pick (product docs, a set of PDFs, a wiki, an FAQ set, API docs) — chunk it, embed it into a vector store. This is your RAG index.
  3. Build a small MCP server exposing two tools: search_kb(query) (vector search → passages + source ids) and get_source(id) (the full source). Your agent must retrieve only through these MCP tools — no direct DB call inside the agent. The MCP boundary is the point.
  4. A deployed frontend takes a question → the agent retrieves via MCP, answers grounded in the retrieved passages, cites the sources, and says "not in my sources" when retrieval is weak.
  5. Make the retrieval visible. Add an inspector — for any question, show the retrieved chunks, their similarity scores, and the source. We want to see the RAG working, not a black box.

Live, public URL, running ≥1 week unattended.

Example — a well-built submission, end to end: You point it at a product's help-center docs (~80 pages). An ingestion step loads them, splits them into ~300-word chunks, embeds each, and upserts them into Qdrant. The frontend has one ask box. We type "What's the refund window?" → your OpenClaw/Hermes agent calls your MCP search_kb, which embeds the question and returns the top 4 chunks with scores; the agent replies "30 days from delivery" with a citation card to the exact passage. Right below, a retrieval inspector lists those 4 chunks, their similarity scores (0.81, 0.74, …) and source headings — so we can see why it answered that. Ask something off-corpus → "I don't have that in my sources." The README redeploys the whole thing (ingestion included) in ~10 minutes. (Going further: hybrid search + a reranker, a confidence threshold on the top score, a coverage view of the corpus.)

LLM + embeddings: the agent needs a model and an embedding model. Ollama Cloud free tier (rate-limited — design assuming it will throttle), or any LLM key you already have.

Ground rules

  1. Retrieve only through MCP. The agent must call your MCP tools to search — a direct DB query in the agent defeats the test and is graded down.
  2. Ground and cite, or abstain. Answer only from retrieved passages, show the source, and say "not in my sources" rather than guess — this gate is graded directly.
  3. Free tiers only. Small corpus, small models, cap tokens.
  4. Keep it running ≥7 days. README must let us redeploy from scratch (corpus ingest included).
  5. Blocked? Tell us. We'd rather unblock you on infra/MCP/credits than have you stuck — just ask.
